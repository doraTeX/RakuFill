# RakuFill（ラクフィル）

フォームの入力状況を URL ごとに名前を付けて保存し、ワンタッチで復元できる Chrome 拡張機能。
保存済みのページを開くと自動で上部バーが現れ、最後に使った入力状況が自動適用されます（OFF 可）。

## 機能

- **保存**: ツールバーのアイコンを左クリック → 上部バーの「保存」→ 名前を付けて現在のフォーム入力状況を保存
- **復元**: バーのプルダウンから保存済みの入力状況を選ぶと即適用
- **自動適用**: 保存データのあるページを開いた瞬間にバーが自動表示され、「最後に使用した入力状況」が自動適用される（ダッシュボードで OFF にできる）
- **ダッシュボード**: アイコンを右クリック →「ダッシュボード」。JSON の直接編集（即時反映・保存ボタンなし）、自動適用の ON/OFF、入力状況の削除・並べ替え・リネーム（✎）・複製（variant 作りに便利）。URL には表示名（愛称）が付けられ、URL・表示名・入力状況名での絞り込み検索もできる
- **フォーム構成の変化に強い**: 保存時から入力欄が消えていてもエラーにせずスキップ。保存後に新設された入力欄はデフォルトのまま触らない
- **クラウド同期**: Chrome にログイン中の Google アカウントで、保存データと設定を端末間同期（`chrome.storage.sync` 利用、デフォルト ON、ダッシュボードで OFF 可）
- **インポート/エクスポート**: ダッシュボードのチェック状況（自動適用・クラウド同期）と全 URL の保存済み入力状況を 1 つの JSON ファイルとして書き出し・読み込みできる（バックアップや、Chrome アカウントを使わない手動移行に）
- UI は日本語（`_locales` ベースなので他言語追加も容易）

Chrome ウェブストアへの公開手順は [docs/PUBLISHING.md](docs/PUBLISHING.md) を参照。

## 開発

```sh
npm install
npm run build   # 型チェック + dist/ へビルド
npm test        # vitest による単体テスト
npm run icons   # アイコン PNG の再生成
```

## インストール（開発版）

1. `npm run build` を実行
2. Chrome で `chrome://extensions` を開く
3. 右上の「デベロッパーモード」を ON
4. 「パッケージ化されていない拡張機能を読み込む」→ このリポジトリの `dist/` を選択

## 動作確認手順

1. `test-page/index.html` を HTTP で開く（例: `python3 -m http.server -d test-page 8000` → http://localhost:8000 ）
   - `file://` で開く場合は拡張機能の詳細画面で「ファイルの URL へのアクセスを許可する」を ON にする
2. フォームに適当に入力 → アイコン左クリック → バーの「保存」→ 名前を付けて保存
3. ページをリロード → バーが自動表示され、入力が自動復元されることを確認（2 秒後に出現する欄にも遅れて適用される）
4. 別の内容でもう 1 件保存し、プルダウンで切り替え適用できることを確認
5. アイコン右クリック →「ダッシュボード」で以下を確認
   - 自動適用チェックボックスを OFF → リロードしても自動適用されない（バーは表示される）
   - JSON を直接編集（値の書き換え）→ ページのプルダウンから適用すると編集後の値が入る
   - 入力状況の「↑ / ↓」並べ替え、「削除」、URL 単位の削除

## クラウド同期の仕組みと制限

- ローカル（`chrome.storage.local`）が正で、`chrome.storage.sync` はそのミラーです。同期処理は background だけが行い、新しい方（最終更新時刻）が勝ちます
- 2 台で同時にオフライン編集した場合など競合時は、**後から更新した側の内容で全体が上書き**されます（マージはしません）
- Chrome 側の制限により同期できる総量は **約 100KB** です（1 アイテム 8KB 制限はチャンク分割で回避済み）。超えるとダッシュボードに同期エラーが表示されますが、ローカルのデータは無事です
- 同期には Chrome へのログインと、Chrome の同期設定で「拡張機能」が有効になっている必要があります

## データの保存場所と注意

- 保存先は `chrome.storage.local`（+ 同期 ON なら `chrome.storage.sync` 経由で Google アカウントに同期）。開発者のサーバー等への外部送信は一切ありません
- **パスワード欄も平文で保存されます。** 共有 PC では注意してください
- 保存対象: `input`（text/email/number/date/checkbox/radio など）、`textarea`、`select`（複数選択含む）
- 保存対象外: `type=file`（ブラウザの制約で復元不可能）、`type=hidden`（CSRF トークン等の復元は危険なため）

## インポート/エクスポートの仕様

- **エクスポート**: ダッシュボードの「エクスポート」ボタンで `rakufill-backup-YYYYMMDD-HHmm.json` をダウンロード。設定（自動適用・クラウド同期の ON/OFF）と全 URL の保存済み入力状況を含む
- **インポート**: 「インポート」ボタンでファイルを選択すると、確認ダイアログの後、**現在保存されているすべてのデータと設定をファイルの内容で上書き**する（マージではなく全置き換え）
- ファイル形式が不正な場合はエラー表示のみで、既存データは変更されない
- ネットワーク通信を伴わないローカルファイル操作

## v1 のスコープ外

- iframe 内のフォーム、`contenteditable`、ページ側 Shadow DOM 内のフォーム

## 構成

```
src/
├─ manifest.ts          MV3 マニフェスト（@crxjs/vite-plugin）
├─ background/index.ts  アイコンクリック・右クリックメニュー・URL 変化の監視
├─ content/
│  ├─ index.ts          自動表示・自動適用のオーケストレーション
│  ├─ bar.ts            Shadow DOM の上部バー UI
│  ├─ capture.ts        フォーム状態の収集
│  ├─ apply.ts          フォーム状態の復元（React 対応のネイティブ setter + リトライ）
│  └─ locator.ts        要素の特定子（id → name → CSS パス）
├─ dashboard/           ダッシュボード画面
└─ shared/              型・storage ラッパー・URL キー正規化・i18n
```
